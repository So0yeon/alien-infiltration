# 🛸 외계인 침투 작전 (Operation Infiltrator)

초등학교 교실에서 얼굴을 보며 토론·추리하는 **실시간 추론 게임**입니다.
웹앱은 진행 보조 도구일 뿐, 활동의 주인공은 교실 토론입니다.

- 👤 **개인정보 없음** — 출석번호(숫자)만 사용. 계정·닉네임·채팅 없음.
- 🔌 **실시간** — Supabase Realtime 으로 교사·학생 화면 자동 동기화.
- ♻️ **재접속** — 학생이 튕겨도 같은 출석번호로 다시 들어오면 그대로 복귀.
- 🧹 **데이터 미보관** — 교사가 "게임 종료"를 누르면 방과 모든 기록이 서버에서 즉시 삭제(cascade).

## 게임 규칙
- 총 5회 임무. **성공 3회 → 인간팀 승리 / 실패 3회 → 침투자팀 승리.**
- 연구원은 임무에서 **성공만**, 외계인은 **성공/방해**를 낼 수 있습니다.
- 임무에 **방해가 1건이라도** 있으면 그 임무는 실패.
- 외계인 수·임무 인원은 참가 인원 기준으로 **자동 계산**되며 교사가 수정할 수 있습니다.

## 기술 스택
React 18 · Vite 5 · Tailwind CSS 3 · Supabase (Realtime + Postgres)

## 1) Supabase 준비 (한 번만)
1. 재사용할 Supabase 프로젝트의 **SQL Editor** 에 `schema.sql` 전체를 붙여넣고 실행합니다.
   - `infil_` 접두사 테이블만 생성하므로 기존 데이터에 영향 없음.
2. 다른 프로젝트를 쓰려면 `.env.example` 를 `.env` 로 복사해 URL/anon key 를 채웁니다.
   (비워두면 코드에 baked 된 기존 프로젝트 사용)

## 2) 로컬 실행
```bash
npm install
npm run dev        # http://localhost:5173
```

## 3) 배포 (Vercel, 1분)
```bash
npm run build      # dist/ 생성
```
GitHub 에 올린 뒤 Vercel → Add New → Project → 저장소 Import → Deploy.
Framework Preset 이 **Vite** 로 자동 인식됩니다.

## 수업에서 쓰기
1. 교사가 전자칠판에서 접속 → **새 게임 시작** → 코드·QR 표시.
2. 학생은 크롬북/휴대폰으로 **QR 스캔 → 출석번호 입력 → 입장**.
3. **역할 배정하고 시작** → 학생들이 각자 정체 확인.
4. 라운드마다: 탐사대 선발 → 승인 투표 → (통과 시) 임무 수행 → 결과 공개.
5. 상단 보드와 탐사대 기록이 칠판에 남아 학생들의 추리를 돕습니다.
6. 끝나면 **게임 종료 · 데이터 삭제**.

> 학생이 중간에 튕기면 교사 화면 상단 **📷 접속 QR** 버튼을 눌러 다시 입장시키세요.
